正文内容
那天晚上我在泡毛尖，电脑屏幕上开着两个网页。左边是 BBC 把 DeepSeek 称作“AI 珍珠港事件”的报道，高可信来源，发布时间 2025 年初；右边是 DeepSeek 官方 API 平台的开发者文档，也属于一手资料。

我盯着那行接口说明，足足愣了有十几秒——**它完全兼容 OpenAI 的格式。**

你敢信？被说成“突袭硅谷”的中国模型，调用的方式跟美国模子一模一样，用着同样的 PyTorch、Transformer 开源栈，跑在英伟达的芯上。那一瞬间，什么“中美 AI 脱钩”的宏大叙事，忽然轻得像茶叶沫子。

**瞧不上的开源，是最粗的贼船缆绳**

后来我顺着线索翻到一篇 Rest of World 今年的深度报道，可信度很高。他们的记者挖得够深：中国短视频平台 TikTok 的推荐算法已经在美国被大量采用；美国研究者堂而皇之地用微软 Azure 训练中国 GPU 跑出的模型；Meta 开源的 Llama 2 呢？被中国开发者二次加工出上百个变种，反过来在 Hugging Face 上霸榜。

Hugging Face 自己 2025 年的社区报告说，最受欢迎的模型库里有 30% 来自中国开发者。我特意去验证了一下，像智谱 AI 的 GLM-4.6V 这种百亿级开源模型，直接摆在上面，Apache 2.0 许可，哪家公司都能商用。而 OpenAI 的 Whisper 语音识别模型，在中国一些提供中文语音转文字服务的小公司手里，用得好好的——OpenAI 官方都没正式进中国。

这些开源协议像看不见的缆绳，把两边的船绑得紧紧的，谁想砍，都得想想自己船上的人会不会先掉进水里。

**封锁线下的活人策略**

当然，芯片出口管制是真的，实体清单也在一轮一轮加长。但技术人很快找到了“活法”。DeepSeek 靠算法优化，用受限的硬件打出了几乎 SoTA 的性能。

最逗的是，我在一个匿名开发者论坛（线索级，不记名不保证真实性）看到一条帖子，一个给硅谷 AI 公司工作的中国工程师说，他们公司明令禁止使用受制裁实体的代码，结果同事直接把 DeepSeek 的权重下到内网做实验，“管理层假装不知道”。

我没法核实这个段子，但放在开源圈的氛围里，一点不违和。我自己也 clone 过好几个 GitHub 仓库，有时候根本不看作者国籍，代码能跑、架构有意思，就本地跑起来看看。

这种“底层的连通”甚至蔓延到了最被忽略的角落。三星工会去年闹着要分享 AI 利润——Rest of World 的报道里提到，这背后是韩国工人生产的 HBM 内存芯片，正嵌在全球英伟达 GPU 的显存里。而中国成熟制程的晶圆代工厂，撑起了全球 AI 终端设备里那些不起眼但必不可少的元器件。扯断任何一环，伤口都会反射到自家身上。

**给“两栖人”的信息差**

这事儿对我这种普通人意味着什么？

我想起公司前阵子选 LLM 做私有化部署的场景。作为一个 IT 技术经理，我当时的脑子是分成两半的：一半要关心技术效果，另一半得盘算——万一未来 OpenAI 对中国断供怎么办？如果客户是外企，它能接受用中国模型吗？

最后我没得选，只能定一个“双源策略”：同时准备美国模型（比如 Llama 3）和中国模型（比如 Qwen2），中间包一层抽象，随时能切。这方案当然不完美，但这是我能想到的、成本最低的现实解法。

而这个“两栖”的活，反而可能是信息差的核心。我能读懂中国 GitHub 仓库里的讨论，也能在 Hugging Face 英文社区里接得上话。在脱钩论的噪音下，这种跨界能力反而被放大了。

我甚至想过一个副业方向：把中国的视频生成模型（像 HiDream）接入美国的 Discord，或者用美国先进的文生图 API 给中亚市场做虚拟试衣。印度一些小团队已经在干了——他们用的就是中美两边的开源模型拼出来的混合方案。

不过，我也知道，这些都是想法。真的下决心去做，我还缺一口气。

**接下来**

我大概率还是会继续研究，但不太会马上跳进去。

今天这篇文章干完，我可能会去追 Rest of World 里提到的那个“两栖开发者”具体怎么赚钱的细节，也可能只是单纯地喝瓶啤酒，打开英雄联盟打一把大乱斗。

我想弄明白的那个问题是：如果极端管制真的发生，个人开发者有没有能力、有没有意愿偷着维持住那根缆绳？但如果这个问题一直没有答案，我可能就一直停在门口，翻翻 paper，看看 GitHub，哪天再在某个论坛上看到一条让我一愣的线索。

然后，就又是一个写得满满、却还没开始行动的周末。

他 5 倍的转化率，就藏在我最怕的那一步

正文内容
周六晚上在奥乐齐买菜，排着队刷手机，看到一个 Reddit 帖子，手就停住了。

一个独立开发者说，他把同一款 AI 产品同时在网站和 WhatsApp 上跑了一圈，结果 WhatsApp 端的用户注册率是网站的 **5 倍**。

5 倍。我当时就想起自己电脑里那个存了三个月的“简历优化助手”文件夹——里面只有一个没写完的 React 首页。

帖子本身说得很模糊，没贴数据，也没说产品到底是啥。Reddit 上老哥的话我打 6 折看（这还是往高了估），但奇怪的是，我一点没觉得离谱。因为我太知道那种冲动——打开聊天框回一句“试试”，比打开浏览器输入网址、等加载、看半天介绍、再点注册，顺畅一百倍。

**注意力的迁移**

那个老哥在帖子里透露，他搭这个 WhatsApp 机器人只用了一个周末，用的是 WATI 这种第三方服务，月费不到 50 美元。他说了一句话，直接戳到我肺管子里：“我后来才想明白，根本不需要那个该死的网站。”

我一个 IT 技术经理，平时一个人扛着整个公司的研发方向，最怕的就是前端——不是说不会，是太耗时间。从选框架到调样式，从部署到兼容，每一步都在提醒我：一个人搞这些，没个头。可如果用户根本不去网站，我花两周搞出来的精美页面，可能还没一条聊天消息管用。

这不是哪个报告告诉我的。respond.io 今年有一份商业指南，说消息应用正在变成用户接触 AI 的第一触点，但这话从他们嘴里出来得打个折——人家就是做这生意的。真正让我感觉到风口的，是 Meta 公开的数据：WhatsApp 全球月活已经超过 20 亿，在很多国家它就是互联网入口。

换句话说，用户已经在聊天列表里了，为什么非要拽他们来我的页面？

**“低摩擦”和“信任偷跑”**

这背后有个行为经济学上的现象，叫“低摩擦效应”。打开 WhatsApp 回复一条消息，大脑几乎没有决策成本，而跳转到陌生网站，光是“这网站安全吗”这一下的迟疑，就足够损失三成潜在用户。

WhatsApp 还偷了一份信任。诈骗和垃圾邮件把用户对陌生链接的警惕拉满了，但来自联系人或者加绿标的企业账号，天然更讨喜。Meta 对 WhatsApp Business API 那套认证机制，放平时我肯定嫌烦，现在想想，那玩意儿其实就是信任的快速通道。

有个 Product Hunt 上推过项目的匿名哥们儿评论说，他拉了 30 个早期测试用户到 WhatsApp 群，一个周末收到 200 条对话反馈，比在网站埋 Hotjar 热力图管用太多。这倒提醒我，我自己也注册过很多 AI 工具，真正高频用的那几个，最后都变成在聊天框里直接交互——比如那个帮我根据需求写 API 文档的 GPTs。

**不做网站的勇气**

说回我自己。那个“简历优化助手”的项目，我研究了至少五个类似的 OpenAI API 调用方案，研究了 WhatsApp Business API 的接入文档，甚至读完了 Twilio 的示例代码。

但就是没动。

每次想开始，脑子里就跳出“还得做网站，还得买域名，还得搞服务器”那套流程，然后所有的热情就被浇灭了。现在回想，这可能是最大的信息差——**不是建更好的货架，而是去人多的广场直接摆摊。**

渠道本身已经是产品的一部分了。就像 ChatGPT 本身就是一个对话框，没人在乎它的“网站首页”长什么样。如果我能把简历优化的功能，直接用对话的方式在 WhatsApp 里交付，为什么还要强迫用户去网页上点按钮？

当然，我也知道风险。把所有数据留在 Meta 的生态里，哪天它改个政策，整个生意可能就塌了。Facebook 平台这几年这种事情还少吗？所以我觉得，这条路更像获取种子用户的冷启动手段，不是千秋万代的护城河。

**接下来呢**

我大概率还是不会马上动手。周末要去河马奥莱囤东西，晚上还得打两把大乱斗。但那个“不用做网站”的想法，像一杯放凉的毛尖，老在舌根那里回甘。

我说“理论上”我准备好了——我知道 API 怎么调，我知道对话流程怎么设计，我知道怎么用 TTL 缓存节省调用次数。下次真的有一个下午的空，我打算先拿自己 2024 年的那份老简历试试，搭个简单的 Telegram 机器人（国内 WhatsApp 用起来太折腾），自己先用用看。

如果连自己都懒得用，那就说明这玩意儿确实没必要往下搞。

但万一，真的挺顺手的呢？

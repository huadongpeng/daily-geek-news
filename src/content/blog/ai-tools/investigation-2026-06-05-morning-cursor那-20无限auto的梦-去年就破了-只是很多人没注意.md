上次写完 GitHub Copilot 百万 token 上下文那篇，有兄弟后台问：Cursor 现在啥情况，Pro 的无限 Auto 是不是还香？

我本来想回“还行”，但打字到一半停下了——因为那个“无限”早在去年就被悄悄换了。

2025 年 6 月，Cursor 官方博客发了篇定价更新。标题没提“取消无限”，说的是“引入使用额度信用池，给予更多灵活性和透明度”。我翻了翻当时的公告，Pro 计划从原来“按速率限制”改成了：每月 $20 固定费，包含 $20 的“前沿模型使用额度”，超额后可以现收现付，按 API 费率走。

（这个变更出自 Cursor 官方博客，可信度算高，但表述相当温柔。）

当时我没怎么上心。直到最近看到 Vantage 的一篇定价解释，又翻到 Dodo Payments 的分析，才重新琢磨了一遍。

真正让我后背一凉的，不是信用池本身，是 Auto 模式。

以前 Pro 用户选 Auto，模型自己决定用哪个，不计次数，只限速率。2025 年 9 月之后，Auto 也开始消耗信用额度，按代币计费。换句话说，原本宣传里的“无限 Auto”，到那时已经没了。

（Vantage 和 Dodo Payments 都是第三方分析平台，有专业度但不代表官方口径，咱们打折看。）

算一笔账。

假设你日常用 Auto 来补全、重构、解释代码，一次复杂请求可能烧掉 $0.1 的代币。$20 的信用池，能撑 200 次左右。如果你一天调个十几回，半个月就见了底。接下来要么停用 Auto 降级到 Tab 补全，要么捏着鼻子付超额费——钱直接走你绑的卡，不设上限。

轻度用户呢？他们一天就用几次，旧规则下被限速，新规则下反而每月可能只用掉 $10，还省了 $10。所以你看，“降价”和“涨价”是同一套方案，看你是谁。

对企业来说更刺激。Teams 计划一开始是“固定请求成本”，后来也改成“基于代币的可变请求成本”。简单任务便宜，复杂任务贵。听着合理，但作为小公司技术负责人，最怕的不是贵，是不可预测。这个月业务忙，代码改得多，下个月账单翻倍，找谁说理去？

我翻了翻 Cursor 的企业定价页，Enterprise 版有集中计费和 SSO，但 Teams 似乎没提供硬性支出上限。小团队如果没人盯仪表盘，月底财务来问的时候，你只能用一句话解释：“AI 是生产力，生产力有时候挺贵。”

（The New Stack 有篇文章标题提到“enterprise spend controls”，但我没读到全文，不确定现在是否已经上线了更强的控制功能，这是个待验证的线索。）

那这事儿对咱们普通人什么判断？

它不是垃圾局，也不是脏机会。Cursor 依然是好工具，但它的代币计费是把 API 成本明牌转嫁给用户。你用得越狠，它赚得越多，而你如果没养成查信用池余额的习惯，就容易超支。

我的做法很简单：打开 Cursor 设置，翻到订阅页，看一眼剩余额度。每次用 Auto 跑一个重活儿，事后瞄一眼消耗。月底前算算还有多少，别被扣了钱才醒。

如果你是副业主，靠 Cursor 出活，可能还得建个简单账本，不然 AI 把利润吃掉一半你都不知道。

一年前那条公告下面，有用户留言：“说好的无限呢？”官方没回。

现在回头看，AI 编码工具的“无限”年代，可能已经结束了。咱们得习惯盯着仪表盘写代码。

我是老花，一个 30 多岁、还在一线摸爬滚打的小公司 IT 技术经理。这里不教成功，只记录我追过的信号、踩过的坑，和我暂时想明白的一点判断。

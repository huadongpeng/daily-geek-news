V2EX 上那个帖子，标题平平无奇：「同事用 AI 疯狂加班，我要不要学？」。点进去，主贴没说出什么，倒是 21 楼的回复把我钉住了。

「我一个同事，用 AI 一个下午生成了 30 多个接口。然后我看了下，有 30% 的功能他自己都不知道干啥用的。」

刷新了两遍，确认没看错。30 多个接口，一个下午，30% 不知道用途。

（贴子来自 V2EX，是一个技术社区，这种个人分享只能当线索，但描述细节太具体，不太像编的。）

我第一反应是羡慕——这效率，赶得上我一周。第二反应是恶心——造出一堆不知用途的代码，最后谁来擦屁股？第三反应是凉意——如果老板觉得这样就是「高效」，我这种还在手写代码的人，是不是该滚蛋了？

顺着这股别扭劲，我翻了一圈资料，发现「vibe coding」这个词已经在圈子里传了很久。简单说，就是你用自然语言告诉 AI 要什么功能，它直接吐代码。V2EX 上不止一个帖子在讨论（还有 1218646、1218676 等），LinkedIn 上有人分享用这招接私活月入几千，Medium 上也有非技术背景的人自称靠 vibe coding 月入 $600。

这些成功案例，我得打折看。Medium 和 LinkedIn 的个人分享，一没审计，二可能是卖课引流。Business Insider 的一篇文章标题暗示「 vibe coding 副业并不容易赚钱」（具体内容被截断，只能算高概率推断）。行业博客 Vibehackers 引用 Statista 的数据说 82% 的开发者每天或每周用 AI 编码工具，这数字看着高，但你得知道 Statista 这类调研多少会有样本偏差。

但有一件事是确定的：vibe coding 正在把编程的门槛拉低。以前写个后台接口，得懂框架、懂数据库、懂认证，现在对着 Claude Code 说一句「给我写个用户登录接口，用 JWT」，它一分钟给你搞定。

门槛一低，内卷就来了。

证据 1 里那个一下午肝出 30 个接口的哥们，就是典型。他可能没想害谁，只是老板盯产出，他就用 AI 堆数量。代码质量？可维护性？管不了那么多。反正肉眼可见的「产出」摆在那。

这种事在小公司特别容易发生。小公司技术负责人（比如我）自己都忙得脚不沾地，根本没精力 code review 每一行。如果手下有人这么玩，等发现的时候，仓库里已经一堆 ghost 代码了。

更麻烦的是，这玩意儿会上瘾。BizInsider 那篇（虽然截断了）说有些开发者过度依赖 AI 生成，自己技术反而退步了。我信。一旦习惯当「需求翻译机」，碰到复杂 bug 或架构问题，你就废了。

但话说回来，一棍子打死也不对。V2EX 另一个帖子（1218676）里，有人说自己 99% 的代码靠 AI 写，年薪还涨了。这让我冷静了一下：vibe coding 本身不是毒药，怎么用才是。

我发现一个规律：用 vibe coding 成功的人，都不是纯「翻译」，而是「导演」。

他们会拆需求、会验收、会组合，更重要的是，他们知道自己要什么。比如证据 5 那个自由职业者，他说接活最重要是和人聊清楚需求，然后再让 AI 生成第一版，自己改改逻辑、调调接口。技术部分反而占比不大。

所以 vibe coding 对普通程序员的影响，得分开看：

如果日常就是增删改查、简单页面、标准接口，那危险了。这种活 AI 确实能干，成本还低，你的价值被稀释。

但如果能做复杂业务设计、跨系统架构、性能优化、安全审查，或者能把 AI 工具管得明明白白，那你不仅不会被替代，效率还能翻倍。证据 8（Vibehackers）里引用的那句「2026 年 vibe coder 年薪中位数 $120k-$180k」虽然可能是营销话术，但方向没错：头部 AI 使用者确实在值钱。

回到副业这块，vibe coding 确实开了一扇窗。以前想做个 SaaS 得组队，现在一个人用 Cursor + Supabase + Claude Code，可能一两个周末就能推出个雏形。

研究笔记里零星提到一些收入范围：从几百美元到两万美元一个月都有，但这些都是自述或平台宣传，真实成功率我打个三折。国内更没看到多少可靠案例。

如果你真想试试，我的判断是这样：

适合谁？有点技术底子的程序员（至少知道 HTTP 和数据库），或者特别懂某个垂直行业（比如你做了十年家具销售，知道经销商缺什么工具）的人。

第一步：别急着买课。用免费版 ChatGPT 或通义千问，试着描述一个你特别清楚的需求（比如「帮我写个简单的库存记录页面，能增减数量，显示当前库存」），看看 AI 生成的代码能不能跑。这花不了你一小时，零元。能跑通，再考虑用 Cursor（$20/月）做个更完整的原型。

危险信号：如果你发现生成的代码你完全看不懂，或者它提醒的坑（比如 SQL 注入）你都不清楚，就别往复杂项目上走，先补基础。还有就是，千万别因为试两个周末成功了就辞职——我这里说的是副业尝试，不是赌命。我见过太多人（包括以前的我）觉得风口来了，一猛子扎进去，最后淹个半死。

最关键的风险是市场饱和。门槛低意味着人人都能搞，很快你会发现自己做的工具别人也做了，而且免费。利润会卷到几乎为零。另外，AI 生成的代码版权和漏洞问题，都是定时炸弹，虽然现在没爆，但爆了可能就是一口锅。

国内玩家的话，支付、备案这些得额外操心。用 OpenAI API 还得翻墙，合规性也悬。替代方案可以是国内模型（通义灵码、百度 Comate），但那种「用嘴写代码」的体验可能打折扣。

至于我自己，我认这个方向，但因为平时工作就是管技术，再用 vibe coding 搞副业，等于全天和代码死磕，别说周末露营，眼睛都得瞎。所以老花我自己，可能会把 vibe coding 当成工作里的提效工具，而不是副业主力。

对于有条件的兄弟，如果你现在有空闲时间，又真有个小痛点想解决，不妨花两个周末试试。优先级是先验证「有没有人需要」，而不是「我开发了多少功能」。那个一下午 30 个接口的前车之鉴，千万记住：功能多不等于有用，有用才值钱。

最后收一下：vibe coding 是一场技能平权运动，也是个体力陷阱。它可能让初级程序员的光环褪色，也可能让有判断力的老手如虎添翼。

现在这阶段，证据还不足以下定论，但警醒已经足够多了：别堆无意义功能，别让自己的技术退化，也别被收入神话忽悠。

先看清楚，再动手。

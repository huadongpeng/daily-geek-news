Twitter图库抓取

目标市场画像#

  • 为什么选Twitter/ X？ 全球月活4.5亿用户，每天产生超过5亿条推文和图片。这些图片天然带有用户真实场景和高互动标签，非常适合用来训练AI视觉模型、生成社交媒体素材，或做竞品视觉分析。个人开发者无需任何成本即可获取海量素材。
  • 适合卖什么？
    1. 定制AI头像生成服务（用爬到的图做风格迁移训练）
    2. 社交媒体图片素材包（按主题打包，例如“科技创业者办公室实拍”）
    3. 电商产品图背景替换服务（用真实街景图做背景）

从零到上线的每一步#

Step 1：注册Twitter开发者账号

  • 访问 developer.twitter.com，点击“Sign up”。
  • 使用个人邮箱注册，推荐用Gmail或Outlook。
  • 填写基本信息：姓名、电话（可选）、国家。
  • 选择“Individual Developer”计划——免费，每月可发送500条推文、读取15000条推文数据。
  • 提交申请表，简述用途，例如“build a photo gallery app”。
  • 审核通常24小时内通过，通过后你会获得以下信息：
    • API Key
    • API Key Secret
    • Bearer Token（推荐用这个）
  • 注意： 如果你的IP在国内，可能需挂梯子，否则注册页面加载缓慢。

Step 2：配置Python环境

  • 安装Python 3.9以上版本，在终端运行：
    pip install requests
  • 创建一个Python脚本 fetch_twitter_media.py

Step 3：编写代码拉取图片 复制以下代码到文件中（替换 YOUR_BEARER_TOKEN）：

import requests
import json

bearer_token = "YOUR_BEARER_TOKEN"

def get_tweet_media(tweet_id):
    url = f"https://api.twitter.com/2/tweets/{tweet_id}?expansions=attachments.media_keys&media.fields=url,preview_image_url"
    headers = {"Authorization": f"Bearer {bearer_token}"}
    response = requests.get(url, headers=headers)
    if response.status_code != 200:
        print(f"Error: {response.status_code}, {response.text}")
        return []
    data = response.json()
    media_urls = []
    if "includes" in data and "media" in data["includes"]:
        for m in data["includes"]["media"]:
            if "url" in m:
                media_urls.append(m["url"])
    return media_urls

if __name__ == "__main__":
    # 找一个包含图片的推文ID，例如某网红发的图
    tweet_id = "1790045219106652160"  # 示例ID，实际请改
    urls = get_tweet_media(tweet_id)
    for url in urls:
        print(url)

运行：python fetch_twitter_media.py，你会在终端看到图片URL列表。

  • 常见坑： Twitter API限制每秒最多300个请求，免费计划每月15000次读取。不要循环请求太快，建议手动脚本执行。

Step 4：存储和使用图片

  • 用Python urllib 或 requests 下载到本地：
    import urllib.request
urllib.request.urlretrieve(url, "photo.jpg")
  • 建议按主题建文件夹（如：ai_art / street_photography）。
  • 用途一：将这些图片输入到Stable Diffusion或Midjourney中做风格训练（需有ComfyUI或SD Web UI）。
  • 用途二：直接打包成图片素材包，在Gumroad上出售（定价2-5美元）。

Step 5：收款

  • 注册 Gumroad（支持中国用户，用PayPal收款，无月费）。
  • 上传图片素材包，设置价格。
  • 买家付款后，平台自动发送下载链接。
  • 提现到PayPal，再转回国内银行卡（手续费约2-3%）。
  • 回款周期：买家下单后7天后可提现。

工具链推荐#

  1. Python requests库（免费）：核心工具，无需安装第三方库。替代品：cURL命令行。
  2. Gumroad（免费，抽成5%）：销售数字商品的最佳平台。替代品：Etsy（需美区身份，较繁琐）。
  3. Stable Diffusion Web UI（免费）：用于风格迁移训练。替代品：ComfyUI（更灵活但复杂）。

避坑清单#

  1. API密钥泄露导致滥用封号： 不要把Bearer Token硬编码在打包脚本里；尽量使用环境变量或配置文件。
  2. 下载受版权保护的图片： 不要爬取品牌官方图或带水印图，建议只爬普通用户分享的生活/街拍图。
  3. 请求频率过高被限： 免费计划每月15000次读取，平均每天500次。如果你一次爬100张图，大概消耗100次配额，足够日常使用。

成本拆解#

  • 启动成本： 0元（开发者账号免费、Python免费、Gumroad免费）
  • 时间投入： 首次配置约2小时，之后每周维护1-2小时
  • 月度运营成本： 0元（除非你买VPS，但本地电脑+外网连接即可）
  • 预期首周收益： 卖出10份5美元素材包，减去Gumroad 5%抽成和PayPal提现费，到手约40美元（约280元人民币）

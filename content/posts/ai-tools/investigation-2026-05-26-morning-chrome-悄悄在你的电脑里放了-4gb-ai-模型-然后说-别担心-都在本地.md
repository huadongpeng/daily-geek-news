## 你的浏览器可能已经变成了 AI 容器，而你毫不知情

2026 年 5 月，大量 Chrome 用户发现自己的磁盘上多了一个约 4GB 的文件 `weights.bin`——这是谷歌的 Gemini Nano AI 模型。它随着 Chrome 148 版本的更新而来，没有弹窗提示，没有选项开关，甚至在用户手动删除后，浏览器会自动重新下载。

“这就像有人在你的房子里悄悄放了一个监听设备，然后告诉你它不会打电话回家。”一位 Reddit 用户如此形容。

## 为什么普通人应该关心

浏览器是我们访问互联网的核心工具，长期以来被视为相对中立、透明的软件。但 Chrome 此举打破了两个基线：**控制权**（用户应有权决定是否安装某个大模型）和**隐私承诺**（谷歌声称所有处理都在本地，但措辞的变化让人生疑）。

据 The Register（高可信）报道，谷歌在 2026 年初修改了 Chrome 的 AI 隐私描述，删除了长期存在的“所有 AI 处理均在本地进行”的明确保证，换成了更模糊的说法。虽然后续澄清处理仍在本地，但这种“先改后解释”的方式加深了不信任。

### 沉默的 4GB 模型从何而来？

多个独立来源（包括 Medium 个人博文、科技新闻网站 GizChina、LinkedIn 用户帖子及 Reddit 讨论）交叉印证：Chrome 148 版本在后台下载 Gemini Nano 模型，大小约 4GB，目的正是为谷歌的 Prompt API 服务。该 API 允许网页开发者直接调用浏览器内置的 AI 功能，例如文本摘要、智能回复和代码生成，无需联网或第三方模型。

- 谷歌官方对 The Register 的声明称：“Gemini Nano 仅在用户设备上运行，不会将数据发送至服务器。”（高可信）
- 然而，用户无法选择不安装该模型。Medium 博主详细记录了多次删除 `weights.bin` 后 Chrome 自动恢复该文件的过程。（中可信，因个人博文但内容详尽且被多源引用）
- 社区普遍反应负面，主要集中于磁盘空间占用、未授权安装和缺乏退出机制。（中-高可信，源于多个社交平台的主流情绪）

### Mozilla 说：这是在毁掉开放的 Web

这一内置 AI 策略最大的反对声音来自 Mozilla。据 The Register 引述，Mozilla 公开批评 Chrome 的 Prompt API，认为将 AI 能力深度绑定到浏览器且不提供标准化的 Web API，会迫使开发者依赖 Chrome 的特定实现，从而损害 Web 的开放性。“如果只有 Chrome 能运行某些 AI 功能，那么网站将只为 Chrome 优化，其他浏览器会被边缘化。”这种担忧不无道理：Chrome 目前占桌面浏览器市场份额约 65%，其行动可以定义事实上的标准。

### 反驳视角：本地 AI 或许是为了保护隐私

谷歌的辩护逻辑是：将 AI 模型放在本地，避免数据上传到云端，恰恰是为了隐私。从技术角度，本地推理确实可以减少数据泄露风险，且 Gemini Nano 是轻量级模型，适合在个人设备运行。部分开发者认为，这比每次请求都调用云端 API 更安全。

但争议的焦点在于 **用户的选择权**。一个类比是：即使一个软件声称某项功能完全离线，它也必须告知用户并允许卸载，而不是悄无声息地塞进更新包。

同时，目前尚无证据表明 Gemini Nano 在用户未使用 AI 功能时会主动运行或扫描网页内容——但也没有证据排除这种可能性，因为模型文件是全局存在的。（待验证线索）

### 这件事会怎样影响你我

对于普通用户，至少三个实际问题：
1. **磁盘空间**：如果你使用 128GB 笔记本且多分区，4GB 可能触发空间焦虑。
2. **浏览器选择**：是否转向 Firefox、Brave 或其他更克制的浏览器？Edge 已宣布禁用 Prompt API，理由是未准备好。
3. **未来预期**：如果 Chrome 可以随意添加大型模型，下一次会是什么？GPT 级别的本地模型？这会让浏览器从“页面渲染器”变成“本地 AI 平台”，而你未必愿意。

对于开发者，Prompt API 可能提供便利，但也带来被谷歌生态锁定的风险。

### 悬而未决的疑问

- 谷歌在欧洲以隐私为名阻止某些 AI 功能，却为何在全球范围内静默部署模型？是否存在双重标准？（Reddit 线索，未证实）
- Mozilla 是否真会实施 AI 切换开关？火狐的这一功能能否如期推出？（目前仅有 Facebook 片段，无时间表）
- 谷歌是否计划将 Gemini Nano 与 Chrome 的其他数据（如浏览历史）结合来进行个性化推荐？官方声明未提及，但技术可能性存在。

**一句话总结：** 静默注入 4GB AI 模型是浏览器权力的重大延伸，而用户连一个“不，谢谢”的按钮都没有。

## 导言

开发者常说“加个工具调用不就多一次 API 请求嘛，能贵到哪去？”但 Augment Code（高可信）的模拟数据显示：一个朴素智能体循环在 10 步后，单次调用的 token 消耗可能膨胀至初始的 4 倍，若每日执行 1000 次，轻量场景月成本可超数百美元，重型文件场景则轻松破千——而这还不算多智能体架构那 15 倍的隐藏税。

## 为什么这件事现在重要？

AI 代理（Agent）已经从概念验证阶段进入生产环境，从代码助手到客服机器人，工具调用是标准配置。但多数开发者只关注模型定价，忽视了调用过程中上下文窗口被不断填充的现实。当代理调用工具获得一个 4KB 的 JSON 响应，即使只用到其中一个字段，整个 JSON 都会被写入上下文，导致后续每次推理都要为这些多余数据付费。

这种增长不是线性的：工具返回的数据越多，未来每一步的成本就越高，形成二次曲线。对于个人开发者或小团队，这可能意味着一个看似智能的代理在无意识中吞噬掉整月预算。

## 证据展开：成本增长的量化与补救

### 1. 朴素循环的真实成本

Augment Code（高可信）构建了一个包含 5 个工具的代码智能体，模拟 10 步循环：
- 在文件密集场景下（每步约 8000 tokens），总消耗约 97k tokens，按 gpt-4o 定价估算约 $1.49。若每天 1000 次调用，月成本约 $1,340。
- 若将上下文窗口限制为仅保留最近 2 轮对话，同样场景成本降至 $0.86，降幅约 42%。轻量场景（每步 1000 tokens）成本绝对值更低，但比例相似。

### 2. 成本的主要来源

MindStudio（中可信）的拆解显示，工具调用成本主要分布在：
- 工具模式定义和参数（一次性成本，可忽略）。
- 工具返回的完整响应数据——即便大量字段无用，也会占用 token 并推高后续成本。

Augment Code 进一步揭示：仓库中的 AGENTS.md 文件若过于详细，会使推理成本增加超 20%，且可能降低任务成功率；应只描述最小必要信息。

### 3. 优化策略实测

- **上下文窗口约束**：如前所述，从 10 步全历史到仅保留最近 2 轮，成本降 42%（Augment Code, 高可信）。
- **提示缓存**：可将固定系统提示部分的成本降低 90%，但对对话历史二次增长无效（Augment Code, 高可信）。
- **MCP 优化**：通过过滤工具响应、执行代码精简数据、使用紧凑编码（如 TOON）等技术，MindStudio 声称最高可削减 98% 的 token 用量，且不影响代理质量（MindStudio, 中可信）。
- **多智能体架构的成本税**：协调器-专家模式虽解耦了逻辑，但整体 token 消耗约为单聊交互的 15 倍，仅在任务价值足以覆盖开销时才合算（Augment Code, 高可信）。

### 4. 待验证的低成本技巧

Reddit 用户（线索级）声称通过“变量传递动态数据”等方式将 token 用量和速度削减约 82%，但缺乏具体实现和对照环境，暂不能作为行动计划。

## 反驳视角：这些数字可能低估或高估

- 所有成本数据均基于特定模拟环境（Augment Code 的代码助手场景、gpt-4o 定价），迁移至其他模型或任务（如客服、数据分析）时，二次增长规律可能存在，具体数字会变。
- MindStudio 的工具优化建议来自其自身工具生态（他们提供低代码代理构建平台），带有商业推广动机，其“98% 削减”的实际可达性需独立验证。
- 多智能体 15 倍税赋是极端情况，若专家智能体任务极简，实际倍数可能更低。

## 影响与悬问

### 对读者的实际影响
- 如果你是正在或准备构建 AI 代理的开发者，立即检查两个点：工具返回的数据是否可过滤至仅必要字段；上下文窗口是否可根据任务阶段动态裁剪。这两项改动几乎零成本，却可能大幅降低月度支出。
- 对于副业项目或独立开发，避免过早引入多智能体架构，除非有明确的量化指标证明任务价值超过那 15 倍的额外开销。

### 最值得继续追问的疑问
目前缺乏来自生产环境、多任务混合的真实成本基准。Augment Code 的研究虽严谨，但仍属模拟。一个横跨不同领域、不同模型、真实业务流量的成本全景图尚未出现，而这才是帮助开发者做出明智架构决策的关键。

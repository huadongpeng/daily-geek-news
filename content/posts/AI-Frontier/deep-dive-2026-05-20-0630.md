### 从注册到产出第一个结果的完整流程

#### 1. 安装Goose

打开终端（Mac/Linux/WSL），运行：
```bash
curl -fsSL https://raw.githubusercontent.com/block/goose/main/install.sh | bash
```
Windows用户使用PowerShell（管理员）：
```powershell
winget install goose
```
无需注册，无需信用卡。

#### 2. 配置OpenAI API密钥（或免费使用本地模型）

Goose默认使用OpenAI API。如果你有API密钥：
```bash
export OPENAI_API_KEY="sk-你的密钥"
```
没有密钥？使用内置本地模型（较慢但免费）：
```bash
goose set-model local
```

#### 3. 运行第一个任务

让Goose创建一个Python爬虫：
```bash
goose "创建一个Python脚本，爬取https://news.ycombinator.com的标题，保存到hacker-news.txt"
```
Goose会自动：
- 安装requests, beautifulsoup4
- 写爬虫代码
- 运行并测试
- 输出结果

#### 4. 查看结果
- 文件hacker-news.txt已经生成
- 终端显示爬取到的标题列表
- 耗时约45秒（本地模型约2-5分钟）

### 进阶技巧：3个大多数人不知道的用法

1. **批量修改文件**：
```bash
goose "把当前目录下所有.py文件中的print换成logging.info，并添加时间戳"
```
自动遍历、修改、备份原文件。

2. **写单元测试**：
```bash
goose "为utils.py中的compute函数写3个单元测试，使用pytest，输出到test_utils.py"
```
自动分析函数签名，生成测试用例。

3. **一键部署到GitHub Pages**：
```bash
goose "将dist文件夹内容部署到GitHub Pages，使用gh-pages分支"
```
自动执行git操作、推送、配置。

### 常见错误和排障

| 错误现象 | 原因 | 解决方法 |
|---------|------|---------|
| "command not found" | 未安装成功 | 重新运行安装命令 |
| "API key not set" | 未配置密钥 | 运行export命令或改用本地模型 |
| 运行超时 | 本地模型太慢 | 使用OpenAI API（充值5美元可用数月） |
| 乱码输出 | 终端编码问题 | 在命令前加 `LANG=en_US.UTF-8` |

## 痛点：智能体开发成本高、依赖云端API

你用AI做自动化任务——比如自动回复邮件、爬取网页数据、整理文档——但每次调用GPT-4/Claude的API，一次任务就要几毛到几块钱。一个月跑几百次，几百块就没了。而且数据全传到云端，隐私也不放心。

本地模型便宜，但之前准确率低得没法用。一个8B的本地模型跑多步骤任务（比如"查天气→发提醒→写总结"），准确率只有53%，一半情况会出错。

Forge解决的就是这个问题：通过自动修复、重试、上下文管理等手段，把本地8B模型的多步骤任务准确率从53%拉到99.3%，比没优化的Claude Sonnet（87.2%）还高。

**启动成本**：0元（已有电脑）或约4000元（新购二手RTX 3060显卡）
**时间投入**：首次部署2小时，后续每次新任务1小时
**预期首周收益**：节省API费用约200元（按每天50次调用算）

## 提效方案：工具链和完整步骤

### 工具链架构

Forge + llama.cpp/llama-server + 本地8B模型（如Ministral-3 8B Instruct）

- **Forge**：GitHub开源框架，负责智能体可靠性（重试、解析修正、上下文管理）
- **llama-server**：本地运行LLM的推理引擎，兼容OpenAI API格式
- **模型文件**：8B参数的GGUF格式模型，约4-6GB

### 从安装到首次运行

#### 第一步：准备环境（Windows/macOS通用，30分钟）

1. 安装Python 3.11+（[下载](https://www.python.org/downloads/)），安装时勾选"Add Python to PATH"
2. 安装Git（[下载](https://git-scm.com/downloads)）
3. 打开终端（Win: 按Win+R输入cmd → 按回车；Mac: 打开"终端"应用）

```bash
# 验证Python安装
python --version
# 应该输出类似 Python 3.11.x

# 验证Git安装
git --version
```

#### 第二步：下载Forge（5分钟）

```bash
# 克隆Forge仓库
git clone https://github.com/antoinezambelli/forge.git
cd forge

# 安装Python依赖
pip install -r requirements.txt
# 如果报错，尝试pip install -r requirements.txt --user
```

#### 第三步：下载本地模型（15分钟）

从Hugging Face下载GGUF格式模型（建议Q4_K_M量化版本，平衡速度和质量）

```bash
# 安装huggingface-hub
pip install huggingface-hub

# 下载Ministral-3 8B Instruct Q4_K_M模型（约5GB）
# 如果你的硬盘空间不足，可下载更小的模型如Qwen2.5-7B Q4_K_M
huggingface-cli download lmstudio-community/Ministral-3-8B-Instruct-GGUF Ministral-3-8B-Instruct-Q4_K_M.gguf --local-dir ./models
```

**替代方案**：如果网络慢或无法访问Hugging Face，可用百度网盘或阿里云盘搜索"Ministral-3 8B GGUF"下载后放在./models目录。

#### 第四步：启动模型推理服务器（10分钟）

下载llama.cpp（原生C++推理引擎，速度快）：

```bash
# 下载llama-server
# Windows: 从https://github.com/ggerganov/llama.cpp/releases下载llama-cpp-xxx-win64-cuda.zip
# 解压后找到llama-server.exe
# macOS: brew install llama.cpp

# 启动服务器（假设llama-server.exe在当前目录）
./llama-server --model ./models/Ministral-3-8B-Instruct-Q4_K_M.gguf --port 8080 --n-gpu-layers 99
```

**关键说明**：
- --n-gpu-layers 99 表示尽可能多的层用GPU加速。如果你的显卡显存<6GB，改为--n-gpu-layers 20
- --port 8080 默认端口，后面Forge会连接这个端口
- 启动后看到"Listening on 127.0.0.1:8080"表示成功

#### 第五步：运行Forge示例任务（5分钟）

在另一个终端窗口（不要关llama-server）：

```bash
# 进入Forge目录
cd forge

# 运行一个示例：让智能体帮你整理桌面文件
# 注意：这会在当前目录创建一个结果文件
python forge.py --task "list all .txt files in my Downloads folder and summarize their first lines" --model-url http://127.0.0.1:8080/v1
```

**预期输出**：
```
[Forge] Task started: list all .txt files in my Downloads folder...
[Forge] Step 1: Listing files...
[Forge] Step 2: Reading file contents...
[Forge] Step 3: Summarizing...
[Forge] Success: Found 12 .txt files, summarized.
[Forge] Result saved to ./output/task_result.md
```

### 核心配置代码


```python
# config.py
MODEL_URL = "http://127.0.0.1:8080/v1"  # llama-server地址
MAX_RETRIES = 3  # 出错自动重试次数
CONTEXT_BUDGET = 4096  # 每步保留最大token数

# 步骤配置：告诉Forge你的任务分几步
STEPS = [
    {"name": "list_files", "prompt": "列出指定目录下所有.txt文件"},
    {"name": "read_content", "prompt": "读取每个文件的前100个字符"},
    {"name": "summarize", "prompt": "用一句话概括这些文件的内容"}
]
```

## 进阶技巧

### 技巧1：多智能体协同

Forge支持子智能体。比如你想监控股票价格：

```python
# 创建一个监控智能体
python forge.py --task "monitor AAPL stock price every 10 minutes" --agent-chain monitor,log,alert
```
这样不用手动刷新，智能体自己循环执行。

### 技巧2：挂Telegram或微信通知

Forge支持输出到webhook。一旦任务失败或完成，自动发消息：

```python
# 在config.py里加一行
WEBHOOK_URL = "https://你的机器人地址/send"
```
这样出错时立刻知道，不用盯着终端看。

### 技巧3：用Forge自动修复代码错误

很多开发者用AI写代码，但生成的代码跑不通。让Forge帮你：

```bash
python forge.py --task "run this Python script, if it errors, fix it and rerun, up to 3 times" --code-file my_script.py
```

它会自动识别错误、重写代码、重新运行，直到成功或达到重试上限。

## 常见排障

### 问题1：llama-server启动后提示"CUDA out of memory"

**原因**：显卡显存不够，比如只有4GB。

**解决**：减少GPU层数：
```bash
./llama-server --model ./models/xxx.gguf --port 8080 --n-gpu-layers 0
```
0表示全部用CPU，速度慢但稳定。或者换更小的模型，比如Qwen2.5-7B-Q3_K_M（约3GB）。

### 问题2：Forge报错"Connection refused"

**原因**：llama-server没启动或端口不对。

**解决**：确保先启动llama-server并确认输出"Listening on 127.0.0.1:8080"。检查config.py里的MODEL_URL是否一致。

### 问题3：任务执行到一半卡死

**原因**：上下文过长导致模型超时。

**解决**：在config.py里减小CONTEXT_BUDGET到2048，或增加timeout：
```bash
python forge.py --task "..." --timeout 120  # 默认60秒，这里改为120秒
```

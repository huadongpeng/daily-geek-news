## 一、技术/工具白盒拆解

### 核心原理
Java值类（Value Classes，JEP 401）是Java语言层面的一种新类型，旨在消除对象标识（identity），使得实例可以被VM压扁（flattened）并内联存储在字段或数组中，从而大幅减少堆内存分配和指针间接引用。与普通类不同，值类实例没有对象头（mark word、klass pointer），且不能作为synchronized的目标。

### 关键API
- `value class Point { int x; int y; }` 类似记录类，但强制无标识。
- 所有字段为`final`，默认不可变。
- 可以重写`equals()`/`hashCode()`，但基于值而非引用。
- 不支持反射创建、序列化等依赖标识的操作。

### 与同类方案对比表
| 维度 | Java值类 | 普通Java类 | Java记录类 | Kotlin内联类
|---|---|---|---|---|
| 内存开销 | 几乎无（内联至栈或数组） | 对象头+字段 | 对象头+字段 | 无（编译期展开）
| 标识 | 无 | 有 | 有 | 无
| 可变性 | 不可变 | 可变/不可变 | 仅不可变 | 不可变
| 反射兼容 | 有限 | 全支持 | 全支持 | 有限
| JVM版本要求 | 待预览（JDK 24+） | All | JDK 16+ | Kotlin Native

## 三、实战接入与影响分析（≥800 字）

### 代码示例：简单的值类性能对比
```java
// 使用JDK 24+ with --enable-preview
import jdk.internal.vm.annotation.ValueCapableClass;
import org.openjdk.jmh.annotations.*;
import java.util.concurrent.TimeUnit;

@ValueCapableClass
final value class Point {
    int x;
    int y;
    
    Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
}

@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.NANOSECONDS)
public class ValueClassBenchmark {
    
    @Benchmark
    public Point createPoint() {
        return new Point(1, 2);
    }
    
    @Benchmark
    public Object[] arrayAccess() {
        Point[] points = new Point[1000];
        for (int i = 0; i < 1000; i++) {
            points[i] = new Point(i, i+1);
        }
        return points;
    }
}
```

### 常见坑点和排错指南
- **坑1**：JDK预览版不支持生产环境部署。
- **解**：明确告知用户仅用于实验，待正式版发布再迁移。

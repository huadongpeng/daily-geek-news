# 80元平板变Debian工作站：零成本双启动指南

## 效率痛点（100字）

你手头有一台闲置的安卓平板（比如Doogee U10），性能跑不动大型App，卖二手不值钱。每天花30分钟纠结要不要买台600元的笔记本跑Linux。一个月下来，你在等待、搜索、搬运文件上浪费了15小时——相当于接外包少赚450元。别扔，它能变成你的便携Debian工作站。

## 自动化/提效方案

### 工具链架构
- 硬件：Doogee U10平板（RK3562芯片，4核Cortex-A53，2GB RAM，16GB存储）
- 系统：Debian 12 Bookworm（SD卡启动，不影响原安卓）
- 构建工具：rkdebian（GitHub上的自动构建脚本）
- 存储：一张32GB以上SD卡（建议U1以上速度）

### 从安装到首次运行的完整步骤

**前置条件**
- 一台能联网的电脑（Windows/macOS/Linux均可）
- 一个SD卡读卡器
- SD卡一张（至少16GB，推荐32GB）
- 约1GB可用硬盘空间

**第一步：下载构建系统**
```bash
# 打开终端，克隆项目
git clone https://github.com/tech4bot/rk3562deb.git
cd rk3562deb
```
**启动成本：0元**
**时间投入：0.2小时**

**第二步：安装依赖**
```bash
# Debian/Ubuntu系统
sudo apt update
sudo apt install -y binfmt-support qemu-user-static gdisk wget xz-utils
# macOS系统需安装Homebrew和QEMU（略慢）
# Windows可用WSL2后执行上述命令
```
**时间投入：0.1小时**

**第三步：构建镜像**
```bash
# 执行构建脚本，自动下载rootfs并打包
sudo ./build.sh
# 等待约10分钟，输出rk3562-debian.img文件
```
**时间投入：0.2小时**

**第四步：写入SD卡**
```bash
# 先用lsblk（Linux）或diskutil list（macOS）确认SD卡设备名
# 假设是/dev/sdb（请仔细确认！别写错盘）
sudo dd if=rk3562-debian.img of=/dev/sdb bs=4M status=progress
sync
# Windows用Rufus或balenaEtcher刷入img文件
```
**时间投入：0.1小时**

**第五步：启动**
1. 将SD卡插入Doogee U10平板（关机状态）
2. 按电源键开机，平板自动从SD卡启动Debian 12
3. 首次启动加载约2分钟，出现Debian桌面（轻量XFCE）
4. 默认用户名：debian，密码：debian

**时间投入：0.1小时**

**恢复安卓**
- 关机，拔掉SD卡，开机——平板回归安卓系统
- 插入SD卡，重启——进入Debian
- 完全无损，双系统

**预期首周收益：0元（但省下买新机费用600-2000元）**

## 进阶技巧（3个大多数人不知道的用法）

### 技巧1：安装中文字体及输入法
```bash
sudo apt install fonts-noto-cjk fcitx5 fcitx5-chinese-addons
# 重启后，在系统设置输入法中选择fcitx5，添加拼音
```

### 技巧2：开启SSH远程访问
```bash
# 在Debian终端执行
sudo systemctl enable ssh
sudo systemctl start ssh
# 查看IP：ip addr
# 即可从电脑SSH到平板（192.168.x.x）
```

### 技巧3：扩展存储挂载
```bash
# 平板剩余eMMC（约10GB）可挂载为数据盘
sudo mount /dev/mmcblk1p2 /mnt/data
# 或者用U盘扩展
```

## 常见排障（3个最容易遇到的问题）

### 问题1：平板黑屏无法启动
**原因**：SD卡写入错误或镜像损坏
**解决**：重新格式化SD卡后再次dd写入，确认校验和一致

### 问题2：WiFi无法连接
**原因**：核显驱动不完整
**解决**：使用USB网线转接头，或等待社区更新驱动

### 问题3：屏幕旋转方向不对
**原因**：默认竖屏
**解决**：在设置Display中调整旋转，或编辑/boot/config.txt添加display_rotate=3

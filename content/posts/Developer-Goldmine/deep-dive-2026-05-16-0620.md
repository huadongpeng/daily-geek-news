## 15k行SCSS文件拆分清理上手指南

**启动成本：** 0元
**时间投入：** 30分钟首轮
**预期首周收益：** 清理后节约每周5小时维护时间，若接外包（50元/小时）= 250元/月

### 效率痛点
假设你是一个前端或全栈开发者，维护一个React项目，里面有一个15k行的SCSS文件，混合了页面样式、组件样式、PrimeReact覆盖、旧版代码、实用类、重复定义。

每次改一个按钮颜色，要搜5分钟才能找到定义。每天花至少1小时在这个文件里查找和修改，一个月浪费20小时。如果接外包开发，机会成本约50元/小时，一个月损失1000元。

### 自动化/提效方案
**工具链：** 使用免费AI助手（如Claude或ChatGPT免费版） + 开源`purgecss` + `scss-analyzer`。

### 从安装到首次运行的完整步骤
1. 安装依赖（已安装Node.js则跳过）：
```bash
# 安装Node.js 18+ (如果还没有)
# 访问 nodejs.org 下载LTS版本

# 全局安装SCSS分析器和清理工具
npm install -g purgecss scss-analyzer
```

2. 分析SCSS结构：
```bash
scss-analyzer --file styles.scss --output structure.txt
```
这会输出一个树形结构，显示选择器嵌套层级、定义位置、重复定义次数。

3. 识别死代码：
```bash
purgecss --css styles.scss --content ./src/**/*.tsx --output dead_report.txt
```
这会标记未被任何React组件引用的样式。

4. 使用AI拆分（ChatGPT免费版/Claude）：
- 打开AI聊天界面
- 粘贴`structure.txt`内容，加上这段提示：
> 我项目里有一个15k行的SCSS文件，请根据选择器名称和上下文建议拆成多个模块文件（例如：_buttons.scss, _overrides.scss, _layout.scss）。输出拆分计划和每个文件的前100行内容。

5. 手动按计划把代码段复制到新文件，用`@import`在主入口引入：
```scss
// main.scss
@import 'base/variables';
@import 'base/buttons';
@import 'components/overrides';
@import 'layout/grid';
```

### 效果量化
- 优化前：修改样式需搜索5分钟 + 改3分钟 = 8分钟/次，每天10次 = 80分钟/天
- 优化后：搜索0.5分钟 + 改2分钟 = 2.5分钟/次，每天10次 = 25分钟/天
- 节省：55分钟/天 = 18.3小时/月 ≈ 如接外包值915元/月

### 进阶技巧
1. **AI注释自动生成**：让AI为每个模块文件添加顶部注释说明用途，如：`// _buttons.scss - 按钮和链接样式`
2. **按组件拆分后使用CSS Modules**：把拆出的模块改为`ComponentName.module.scss`，只在该组件内生效，避免全局污染。
3. **增量清理**：每天只拆一个模块，使用`git diff`检查改动，避免一次性出大问题。

### 常见排障
1. **purgecss误删动态类名**：在配置的`content`数组里加上动态字符串匹配，如`'class.includes("btn-")'`。
2. **SCSS变量文件找不到**：拆分后确保所有模块文件的第一行是`@import 'variables';`。
3. **AI建议的划分不合理**：坚持“按功能区域分”，而不是按文件大小，每组不超过200行。

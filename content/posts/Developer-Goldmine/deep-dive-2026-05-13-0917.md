## 免费AI SEO审计指南：让网站被AI搜索看见

如果你有个人博客或接外包项目，AI搜索引擎（如Google AI Overviews、Perplexity）正在成为主要流量入口。但多数人不知道自己的网站在AI眼中长什么样——可能内容正确但没被索引，结构混乱被忽略。手动检查每个页面需要每天花30分钟做重复操作，一个月浪费15小时。

**工具链架构**：
- **Free AI SEO Auditor**（开源）：扫描网站，生成AI搜索友好度报告
- **Ahrefs免费SEO检查器**（可选）：补充关键词和反向链接数据

**从安装到首次运行的完整步骤**：

1. **获取工具**：打开Free AI SEO Auditor的GitHub仓库（来源：ProductHunt），点击“Code”按钮，选择“Download ZIP”或者直接克隆：

```bash
git clone https://github.com/your-fork/ai-seo-auditor.git
cd ai-seo-auditor
```

2. **安装依赖**：确保你有Python 3.8+，运行：

```bash
pip install -r requirements.txt
```

3. **运行审计**：

```bash
python3 auditor.py --url https://你的网站.com
```

4. **查看结果**：工具会在当前目录生成一个audit_report.html，用浏览器打开：

```bash
start audit_report.html  # Windows
open audit_report.html   # macOS
```

报告包含：页面加载速度、结构化数据缺失、关键词覆盖度、AI索引状态。

### 进阶技巧

1. **批量扫描站点地图**：如果网站有sitemap.xml，用 `--sitemap https://你的网站.com/sitemap.xml` 参数扫描所有页面

2. **自定义AI引擎指标**：修改 `config.yaml` 添加你目标AI平台（如BizBot、Perplexity）的特性要求

3. **自动生成优化建议**：运行后工具会输出markdown文件，直接粘贴到项目工单中

### 常见排障

**问题1：扫描报错“timeout”**
解决方法：网站响应慢，增加超时参数：`python3 auditor.py --url https://你的网站.com --timeout 30`

**问题2：报告空白**
解决方法：确保网站公开可访问，不要在本地或需要登录的页面运行；如果用WSL或虚拟机，确认网络正确

**问题3：依赖安装失败**
解决方法：创建虚拟环境 `python3 -m venv venv && source venv/bin/activate`，再重装依赖
